# Kerf boolean readiness matrix

Generated by `cargo run --example readiness_matrix -p kerf-brep`. The matrix
runs every `(primitive A, primitive B, op ∈ {union, intersection, difference})`
combination through `Solid::try_{union, intersection, difference}` and reports
pass/fail.

Failures fall into four root-cause buckets:

1. **M11 phase B — interior endpoint with no boundary anchor.** A closed
   polygon of intersections lies entirely in a face's interior (e.g. a
   cylinder piercing a box face). Currently unimplemented; needs ring
   operators (kemr / kfmrh — M4b) and a stinger-edge phase that grows
   interior vertex chains from the boundary.

2. **stitch: edge has 1 half-edge.** The kept-face graph after classification
   leaves an edge that should be shared by two kept faces but is only
   referenced by one. Often a result of OnBoundary-rule mismatches between
   the two solids; M28 fixed the most common variant.

3. **stitch: edge has 3 half-edges.** Three kept faces share one canonical
   edge — usually a coplanar overlap that's been triple-counted. Needs a
   coplanar-face dedup pass before stitching.

4. **stitch: zero-length edge.** A vase or revolve apex contributes an edge
   whose two endpoints coincide after dedup. Either the polygon for that
   face is degenerate or the input has co-located vertices. Needs the
   stitch to skip zero-length edges (or earlier dedup at face_polygon).
